บทที่ 3 Web/Mobile applications

Web applications" หรือ "Mobile applications" เป็นซอฟต์แวร์ที่ถูกสร้างขึ้นเพื่อให้บริการหรือให้ประสิทธิภาพบนโปรโตคอลเว็บ (สำหรับ Web applications) หรือบนอุปกรณ์เคลื่อนที่เช่น สมาร์ทโฟนและแท็บเล็ต (สำหรับ Mobile applications) ซึ่งเป็นการใช้เทคโนโลยีสารสนเทศเพื่อตอบสนองความต้องการของผู้ใช้

Web Applications: Web applications คือซอฟต์แวร์ที่ทำงานบนเบราว์เซอร์ของผู้ใช้ ส่วนมากมีการใช้เทคโนโลยีเว็บที่ได้รับการพัฒนา, เช่น HTML, CSS, JavaScript ผู้ใช้สามารถเข้าถึงและใช้งานผ่านเบราว์เซอร์จากทุกที่ที่มีการเชื่อมต่ออินเทอร์เน็ต

Mobile Applications: Mobile applications (Apps) คือซอฟต์แวร์ที่ถูกออกแบบและพัฒนามาเพื่อให้ทำงานบนอุปกรณ์เคลื่อนที่, เช่น สมาร์ทโฟนและแท็บเล็ต ส่วนมากถูกพัฒนาโดยใช้ภาษาโปรแกรมที่เหมาะสมสำหรับแพลตฟอร์มนั้น ๆ (เช่น Swift หรือ Objective-C สำหรับ iOS, Java หรือ Kotlin สำหรับ Android)

Model-View-Controller (MVC) เป็นรูปแบบการออกแบบซอฟต์แวร์ที่ใช้ในการสร้างและออกแบบโครงสร้างของแอปพลิเคชัน (Application) หรือระบบซอฟต์แวร์เพื่อให้มีการจัดการกับข้อมูล (Model), การแสดงผล (View), และการควบคุม (Controller) โดยมีวัตถุประสงค์ในการแยกแยะและทำให้ง่ายต่อการบำรุงรักษาและพัฒนา มี
1.Model (ข้อมูล): เก็บข้อมูลและรับผิดชอบในการประมวลผลข้อมูล. นอกจากนี้, Model ยังสามารถแจ้งเตือน View หรือ Controller เมื่อข้อมูลเปลี่ยนแปลง.
2.View (มุมมองหรือแสดงผล): เป็นส่วนที่เกี่ยวข้องกับการแสดงผลข้อมูลที่ Model มีให้เห็น. จะแสดงข้อมูลในรูปแบบที่เหมาะสม (เช่น ตาราง, กราฟ, หน้าจอ).
3.Controller (ตัวควบคุม): รับข้อมูลจากผู้ใช้และสั่งการให้ Model เปลี่ยนแปลงตามความต้องการ. Controller ทำหน้าที่เชื่อมต่อระหว่าง Model และ View.

แนวทางที่อยากประกอบอาชีพ อยากเป็น Dev เพราะว่าชอบในการเขียนโปรแกรมและพัฒนาตัวเองไปเรื่อยๆ และชอบในรายได้ที่ได้รับ
