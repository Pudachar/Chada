สรุปความรู้ที่เรียน

บทที่ 1 Evolution in Software Engineering

1.Waterfall Model (โมเดลน้ำตก): โมเดลน้ำตกเป็นโมเดลที่เริ่มต้นด้วยการทำความเข้าใจความต้องการและการออกแบบทั้งหมดก่อนที่จะเริ่มการพัฒนา. กระบวนการจะเดินหน้าเป็นลำดับขั้นตอนแบบเชื่อมต่อ, และไม่สามารถย้อนกลับไปทำขั้นตอนที่ผ่านมาได้

ข้อดี Waterfall Model
แบ่งงานยากให้เป็นงานที่เล็ก ง่ายต่อการจัดการ มีการกำหนดProductที่ต้องส่งมอบในแต่ละงาน อย่างชัดเจน
ข้อเสีย Waterfall Model
ลูกค้าเห็นและทดลองใช้ Software ก็ต่อเมื่อถึงขั้นตอนสุดท้าย หากมีบางอย่างที่ไม่ตรงกับความต้องการของลูกค้า การแก้ไขยาก แพง เสียเวลา



2.Incremental Model (โมเดลการเพิ่มขึ้น): โมเดลนี้มีลักษณะคล้ายกับโมเดลรอบ แต่กำหนดให้แต่ละรอบนำเข้าฟังก์ชันที่เพิ่มขึ้น

3.Spiral Model (โมเดลห่วงโซ่): โมเดลห่วงโซ่นำเสนอกระบวนการการพัฒนาที่เหมือนกับการบินห่วงโซ่. มีการวนรอบซ้ำ ๆ ตลอดโครงการ เพื่อให้มั่นใจได้ว่าความเสี่ยงถูกจัดการไปได้อย่างเหมาะสม

4.Agile Model (โมเดลแอ็กไจล์): โมเดลแอ็กไจล์เน้นการพัฒนาแบบทันสมัยและทำให้ทีมสามารถตอบสนองต่อการเปลี่ยนแปลงได้รวดเร็ว. มันให้ความสำคัญกับการสื่อสารในทีมและการทำงานร่วมกัน

5.DevOps Model (โมเดลเดฟอปส์): DevOps เป็นการผสมผสานระหว่างการพัฒนาและการดำเนินการ (operations). มีเป้าหมายในการลดเวลาที่ใช้ในการพัฒนาและการนำเสนอซอฟต์แวร์
